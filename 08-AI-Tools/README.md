## Question: Have you used GitHub Copilot, Cursor, Claude Code, Codex or Gemini CLI to improve DevOps productivity? Give a practical example.

## Answer:

Yes, I have used GitHub Copilot to improve my productivity during Devops tasks.

In my previous projects, I used GitHub Copilot while writing Jenkins pipelines, Kubernetes YAML manifests, and Terraform configuration files. Instead of writing everything from scratch, Copilot suggested pipeline stages, Kubernetes resource definitions, and Terraform resource blocks.

For example, while creating a Jenkins pipeline for deploying an application, Copilot suggested the syntax for stages such as checkout, build, Docker image creation, image push to container registry, and Kubernetes deployment. I reviewed and modified the generated code to match our project requirements before committing it.

I also used Copilot to generate Kubernetes Deployment and Service YAML templates, which reduced repetitive manual work. I always validated the generated code before using it in production.

My role: Writing Jenkins pipelines, Kubernetes manifests, and Infrastructure as Code.

Outcome: GitHub Copilot reduced the time spent writing repetitive code and configuration files.
